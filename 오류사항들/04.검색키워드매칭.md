# 오류 보고서: 단순 임베딩 벡터 검색의 식별자(고유 설비명) 매칭 한계 및 하이브리드 검색 도입

## 🚨 문제 상황 (Problem)
기존 DB 오염을 초기화하고 RAG의 검색 범위(`k`) 수를 늘렸음에도 불구하고, AI가 여전히 다른 장비(예: G-101) 문서를 가져오거나 "T-101에 대한 매뉴얼을 찾을 수 없습니다"라고 답하는 등 **고유 설비 번호 검색에 실패하는 현상**이 지속됨.

## 🔍 원인 분석 (Cause)
이는 RAG 시스템에서 흔히 발생하는 **Dense Retrieval(의미 기반 벡터 유사도 검색)의 치명적인 한계점**임. 
현재 ChromaDB에 적용된 임베딩 모델(HuggingFace)은 텍스트의 "의미와 문맥"을 숫자로 변환하여 거리를 계산함. 

* **벡터 검색의 맹점:** AI 모델의 임베딩 관점에서 `T-101`, `T-102`, `G-101` 같은 부품 고유 번호들은 단순히 "어떤 기계의 식별자"라는 **매우 비슷한 의미적 벡터값**으로 취합됨.
* 따라서 사용자가 명확히 "T-101"을 지정해 질문하더라도, 검색 엔진은 "기계 부품, 온도, 상태 이상"이라는 문맥만 비슷하다면 다른 설비 문서(G-101)를 1등, 2등으로 뽑아오는 오류(의미론적 뭉뚱그림)를 범하게 됨.
* 현업(엔터프라이즈 RAG)에서는 고유명사나 시리얼 넘버를 찾기 위해 단순 벡터 검색만 사용하지 않음.

## 🛠️ 해결 조치 (Solution)
벡터 검색의 "유연성"과 키워드/정규식 검색의 "정밀성"을 결합한 **하이브리드 재정렬(Hybrid Re-ranking)** 기법을 로직 내에 직접 구현하여 도입함.

1. **검색 모수 대폭 증가:** 일단 벡터 검색(ChromaDB)으로 가장 문맥이 유사한 문서 후보군을 넉넉하게 10개(`k=10`) 끌어옴. (1차 그물망)
2. **정규표현식(Regex) 필터링:** 파이썬 `re` 모듈을 사용해, 사용자의 자연어 질문 속에서 **[대문자 1~2개 + 하이픈 + 숫자 3개]** 형태의 고유 설비 번호(예: `T-101`, `HX-102`)를 정밀하게 추출함.
3. **최상단 본문 재정렬 (Re-ranking):** 확보한 10개의 문서 중에서 본문(`page_content`)에 추출된 고유 번호(예: 'T-101')가 **정확하게(Exact Match) 텍스트로 들어있는 문서만 1순위로 맨 앞으로 끌어올리고**, 없는 문서는 뒤로 밀어냄.
4. 재정렬된 가장 완벽한 3개의 문서만 최종적으로 LLM(Agent)에게 전달하여 환각과 혼동을 원천 차단함.

## 💻 코드 수정 내역 (Code Changes)

### [Before] 수정 전 코드 (단순 벡터 검색 의존)
```python
# 1. 단순히 상위 K(5)개의 유사 문서만 검색
rag_retriever = vectorstore.as_retriever(search_kwargs={"k": 5})

@tool
def search_maintenance_manual(query: str) -> str:
    # 2. 문맥상 가장 비슷한 문서를 뽑아 그대로 AI에게 넘김 (T-101과 G-101을 구별하지 못함)
    docs = rag_retriever.invoke(query)
    return "\n".join([doc.page_content for doc in docs])
```

### [After] 수정 후 코드 (Hybrid Re-ranking 적용)
```python
import re # 정규표현식 모듈 추가

# 1. 넉넉하게 1차 그물망(10개)으로 후보군 확보
rag_retriever = vectorstore.as_retriever(search_kwargs={"k": 10})

@tool
def search_maintenance_manual(query: str) -> str:
    # 1. 벡터 검색 (의미 기반) - 10개 확보
    docs = rag_retriever.invoke(query)
    
    # 2. 정규표현식(Regex)을 이용한 핀셋 추출 
    # 패턴: 영대문자 1~2개 + '-' + 숫자 3자리 (예: T-101, HX-105)
    equip_id_match = re.search(r'[A-Z]{1,2}-\d{3}', query)
    
    if equip_id_match:
        target_id = equip_id_match.group() # 추출된 'T-101'
        
        # 3. 하이브리드 재정렬 (Re-ranking)
        # 10개 문서 본문을 스캔하여 'T-101'이라는 글자가 100% 일치하는 것만 뽑음
        exact_match_docs = [doc for doc in docs if target_id in doc.page_content]
        other_docs = [doc for doc in docs if target_id not in doc.page_content]
        
        # 키워드 일치 문서를 무조건 최상단(1순위)에 배치하고 나머지는 뒤로 이어붙임
        sorted_docs = exact_match_docs + other_docs
        
        # 최종적으로 가장 완벽한 상위 3개만 AI에게 전달하여 혼란 방지
        final_docs = sorted_docs[:3]
    else:
        # 고유 번호가 아예 없는 일반적인 질문일 경우 (기존대로 3개만)
        final_docs = docs[:3]
        
    return "\n\n".join([doc.page_content for doc in final_docs])
```
