# 오류 보고서: AI 모델의 환각(Hallucination) 현상 발생 및 조치

## 🚨 문제 상황 (Problem)

AI 챗봇에게 특정 설비(예: T-101 가스터빈)의 현재 온도를 제시하며 조치를 물었을 때, AI가 매뉴얼에 존재하지 않는 **가상의 데이터("정상 운전 온도: 60~75℃")를 스스로 지어내어(환각) 대답하는 현상**이 발생함.

**[재현 예시]**

- **User:** "T-101 온도가 79.6도인데 이거 어떻게 조치해?"
- **AI:** "매뉴얼의 상세 기술 데이터시트에 따르면 '정상 운전 온도: 60~75℃'로 명시되어 있습니다. 따라서 현재 온도 79.6℃는 비정상입니다."
- **실제 문제:** RAG의 베이스가 되는 `turbine_manual.txt`에는 애초에 "60~75℃"라는 온도 스펙 수치가 단 한 줄도 적혀 있지 않았음. AI가 인터넷에 학습된 가스터빈의 통상적인 스펙을 가져와 **자신의 뇌피셜로 소설을 쓴 것**임.

## 🔍 원인 분석 (Cause)

RAG(Retrieval-Augmented Generation) 시스템은 문서에 없는 내용을 명확하게 '모른다'고 대답해야 하지만, 주어진 문서(프롬프트)에 문제 판단의 "기준치(정상 범위 스펙)"가 전혀 없을 경우, LLM의 특성상 그럴듯한 답변을 생성하려는 압박에 의해 허위 정보를 생성하는 **환각(Hallucination) 오류**가 쉽게 발생할 수 있음.
결론적으로 기계에 대한 "판단 가이드라인 데이터"가 매뉴얼 내에 명시적으로 결여되어 결함이 발생.

## 🛠️ 해결 조치 (Solution)

RAG 시스템의 환각 오류를 통제하는 가장 확실한 방법은, 검색 대상이 되는 **원천 데이터(매뉴얼 텍스트) 안에 조치의 명확한 기준값(가드레일)**을 삽입하는 것임.
10개의 장비에 대한 가상 데이터를 생성하는 파이썬 스크립트(`generate_mock.py`)를 수정하여, 각 기기별 매뉴얼 텍스트가 생성될 때 온도, 압력, RPM 등 센서의 **[정상 작동 기준 범위]**가 반드시 명시되도록 코드를 업그레이드 하였음. 이후 문서를 재생성하고 RAG 파이프라인의 Vector DB를 재학습시킴.

```python
        # 1. 상단에 설비별 센서 타입 및 정상 범위를 명시적(Explicit)인 규칙으로 정의

            # 중략...
        ]

        # (중략) ...
        # 매뉴얼 텍스트 작성 부스팅
        manual_entry = f"[{eq_name} ({eq_id}) 유지보수 매뉴얼]\n"
        manual_entry += f"- 개요: 1구역에서 작동하는 {type_name} 설비입니다.\n"
      
        # ✅ [해결 로직 추가] 
        # 매뉴얼 상단에 무조건 설비별 '정상 범위 스펙'을 박아넣어 AI가 다른 소리를 못하게 통제
        manual_entry += "- [정상 작동 기준(기술 데이터시트)]:\n"
        for m_key, m_name, m_range in metrics:
             manual_entry += f"  * 정상 {m_name}: {m_range}\n"
      
        # 가이드라인 구문 수정 (기준 범위를 벗어날 경우에만 행동하도록 제한)
        metric_keys = [m[0] for m in metrics]
        if "temperature" in metric_keys or "temperature_in" in metric_keys:
            manual_entry += f"- [고온 경고 조치]: 현재 온도가 정상 기준을 벗어났을 경우 즉시 작동을 중지하고 ERP 시스템 상태를 '점검 요망'으로 변경하세요.\n"
```

## 🚨 추가 문제 상황 (Case 2: 정보 부재 시 허위 사실 날조)

AI에게 "최근 점검일이 3월 15일인 장비들이 뭐야?"라고 전체 장비 모니터링을 질문했을 때, 실제 시스템 데이터를 열어보지도 않고 **"2024년 3월 15일에 점검한 설비는 T-101, P-101, V-102 입니다."**라고 날짜와 연도를 마음대로 지어내어 답변하는 심각한 환각 오류 발생.

## 🔍 원인 분석 (Cause 2)

1. **Tool(도구)의 부재:** AI에게 쥐여준 기능이 "특정 설비 1개의 상태만 조회(`get_equipment_status`)"하는 단일 검색 도구뿐이었음. 전체 목록을 조회할 수 있는 눈(권한/도구)이 없으니 데이터를 읽을 방법이 차단됨.
2. **가드레일(System Prompt) 부재:** 원하는 데이터를 찾지 못했을 때 "모른다"고 답변하게 만드는 강제 지침(시스템 프롬프트)이 없었음. AI는 대답을 해야 한다는 강박관념(LLM 본능) 때문에 인터넷 사전 학습 데이터를 바탕으로 가장 그럴듯한 거짓말을 만들어냄.
